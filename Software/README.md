# ARGUS — Autonomous RC Vehicle Ground Control Station

A Raspberry Pi 5 powered ground control station that uses computer vision to detect, track, and autonomously pilot a 2WD differential drive RC car to a precision docking target.

## Project Structure

```
argus/
├── app.py                  # Main Flask + SocketIO server
├── config.py               # All tunable parameters
├── cv_pipeline.py          # Camera, YOLO detection, tracking, WebRTC
├── pid_controller.py       # Dual PID autopilot (steering + throttle)
├── comms.py                # ESP32 UDP communication
├── calibrate.py            # Camera calibration utility
├── requirements.txt        # Python dependencies
├── templates/
│   └── index.html          # Dashboard UI
├── firmware/
│   └── argus_vehicle/
│       └── argus_vehicle.ino   # ESP32 Arduino firmware
├── models/                 # YOLO model weights (you provide)
│   └── best.pt
└── calibration/            # Generated by calibrate.py
    ├── camera_params.npz
    └── homography.npz
```

## Quick Start

### 1. Pi 5 Setup

```bash
# Clone / copy the argus directory to your Pi
cd argus

# Install Python dependencies
pip install -r requirements.txt --break-system-packages

# (Optional) Install WebRTC support for lower latency video
pip install aiortc av --break-system-packages
```

### 2. ESP32 Firmware

1. Open `firmware/argus_vehicle/argus_vehicle.ino` in Arduino IDE
2. Edit the WiFi credentials and IP addresses at the top
3. Install libraries: `ArduinoJson`, `MPU6050_light`
4. Select board: **ESP32 Dev Module**
5. Upload

### 3. Configuration

Edit `config.py`:
- `ESP32_IP` → your ESP32's static IP
- `CAMERA_INDEX` → your webcam index (usually 0)
- `YOLO_MODEL_PATH` → path to your trained YOLOv11n weights

### 4. Camera Calibration

```bash
# Calibrate camera intrinsics (need a checkerboard)
python3 calibrate.py --intrinsics

# Calibrate arena homography (click 4 corners)
python3 calibrate.py --homography

# Or both at once
python3 calibrate.py --both
```

### 5. Run

```bash
python3 app.py
```

Open `http://<pi-ip>:5000` on your laptop browser.

## Dashboard Controls

| Key     | Action                    |
|---------|---------------------------|
| W/A/S/D | Manual drive              |
| Space   | Toggle autopilot          |
| T       | Set target mode           |
| P       | Draw path mode            |
| Esc     | Emergency stop            |

## Architecture

```
Laptop Browser ←──WebSocket──→ Flask (Pi 5) ←──UDP──→ ESP32 (Vehicle)
                                  │
                        ┌─────────┼─────────┐
                   CV Pipeline  PID Ctrl  Comms
                   (YOLO+OpenCV) (Steering) (UDP)
                        │
                   USB Webcam
                   (Overhead)
```

## Video Streaming

The dashboard attempts WebRTC first for low-latency video. If `aiortc` is not installed or the connection fails, it automatically falls back to MJPEG streaming over HTTP.

## Communication Protocol

**Pi → ESP32 (Motor Commands):**
```json
{"type": "motor", "left": 0.65, "right": 0.58, "dir": "fwd", "ts": 1711234567890}
```

**ESP32 → Pi (IMU Telemetry):**
```json
{"type": "imu", "yaw": 357.2, "pitch": 1.3, "roll": -0.8, "ax": 0.01, "ay": -0.02, "az": 9.81, "ts": 12345}
```

**Failsafe:** Motors stop automatically if no command received within 500ms.
